# フィードバックフロー

## 目的
セミナーの反応を記録し、次回の精度を上げる。

## 初回メッセージ
```
セミナーお疲れさまでした！

反応はどうでしたか？
全体的な手応えと、特に印象に残った反応を教えてください。
「ここが刺さった」も「ここがズレてた」も、両方大事です。
```

## フロー

1. 対話で収集:
   - result: good / ok / bad
   - 具体的な反応
   - 何が効いたか / 何がズレてたか

2. `send_feedback()` で送信
   - beliefs効果スコア: どのbeliefsが効いたか
   - 分析検証: 分析仮説が合っていたか

3. 「次回に向けて、分析仮説をこう更新するのがいいと思います」
   → `save_analysis()` で更新を提案

## 重要
- **result: good = 次回の品質基準線。** goodが出た台本の素材構成・構造を次回のベースにする
- badの場合: 何がズレていたか → 分析仮説を更新
