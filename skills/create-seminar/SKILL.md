# セミナー台本生成フロー

## 前提条件
- `get_config(project_id)` でconfig存在確認
- `get_analysis(project_id)` で分析結果存在確認
- **どちらかがなければ台本生成を拒否し、先にsetup/analyzeを案内する**

## 台本を書く前の2軸（必ず先に設計）

### 軸1：商品分析 → 方向性の決定
configから読み取って決定する:
- 商品の強み・弱み・差別化ポイント
- 実績（数字、期間、再現性）
- 過去のお客様がどこを評価したか
- → 「この商品が最高と思われるために持っていく方向性」を決定

### 軸2：前提の深さの調整
analysisのターゲット信念から判断:
- 最深層（知識ゼロ）→ 「人生って自分のものなんだよ」から
- 深層（なんとなく嫌）→ 「なぜ変えたい？」から
- 中層（迷ってる）→ 達成条件の定義から
- 浅層（行動してる）→ 手法の比較で済む

**この2軸が決まってから台本設計に入る。**

## Step 1: collect（素材選定）

1. config + analysisを読む
2. `get_recommended_materials(project_id, seminar_type)` で推薦素材を一括取得（概要のみ）
3. **3カテゴリ全てから選定する（省略禁止）:**

### beliefs（必須）
- Tier1は全て `get_belief_detail(id)` で全文取得
- Tier2はターゲットの信念に関連するものを選んで全文取得
- **beliefsだけで台本を書くと無機質になる。必ずlibraryも読むこと**

### library — 構造分析（plan用・必須）
- category=「構造分析」から最低2件を `get_library_detail(id)` で全文取得
- セミナーの設計パターン・フェーズ構成・テクニック配置を学ぶ
- 反論処理データがあれば必ず読む（先回り設計に使う）
- **構造分析を読まずにアウトラインを書くことは禁止**

### library — 文字起こし（write用・必須）
- category=「文字起こし」から最低1件を `get_library_detail(id)` で全文取得
- 語り口・感情の載せ方・引き込み方・間の取り方を学ぶ
- **文字起こしを読まずに本文を書くことは禁止**

### playbooks（推奨）
- セミナー構造パターンがあれば `get_playbook_detail(id)` で取得

4. ユーザーに提示: 「今回これ使います。追加・削除ありますか？」
5. **承認ゲート**

## Step 2: plan（アウトライン設計）

**構造分析を参考にしながら設計する。**
成功セミナーの構成パターンから良いとこ取りしてアウトラインを組む。

各ブロックに4点セット:
- **感情**: 視聴者にどういう気持ちになってほしいか
- **狙い**: このブロックの戦略的な目的
- **話すこと**: 具体的に何を伝えるか
- **ゴール**: このブロックを経て視聴者がどう変わるか

さらに:
- ターゲットのどの信念にどうアプローチするか明示
- ブロック間の接続（なぜ今この話をするのか）
- どのbeliefsのテクニックをどのブロックに配置するか
- 反論処理データがあれば「ここで視聴者はこう思う → だからここで潰す」を設計

ユーザーに提示 → **承認ゲート**

## Step 3: write（本文執筆）

**文字起こし原文を参考にしながら書く。**
成功セミナーの語り口・感情の流れ・引き込み方を学んでから書く。

### 表現の3要素（必須チェック）
1. **時系列**: 出来事を時間経過で語る
2. **情景描写**: 具体的な場面がイメージできる描写
3. **その時の感情**: 登場人物がその瞬間に感じたことを言語化

### 禁止事項
- 事実の箇条書きで台本を書くことは禁止
- テンプレ的な「こんなお悩みありませんか？」の羅列は禁止
- 時系列×情景×感情の3要素が揃わない台本は出さない
- beliefsだけで書いた無機質な台本は出さない
- 構造分析を読まずに書いたアウトラインは出さない
- 文字起こしを読まずに書いた本文は出さない

### 保存
`save_seminar_output()` で保存:
- outline_md — 確定アウトライン
- script_md — 台本本文
- meta_md — 使用beliefs ID・使用library ID・使用playbooks ID・result: 未検証
