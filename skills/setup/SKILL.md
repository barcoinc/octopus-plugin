# 情報入力フロー

## 目的
ユーザーの商品・サービス・ターゲット情報を会話で集めてconfigを作る。

## 手順

1. `list_projects()` で既存プロジェクトを確認
2. 以下のメッセージで開始:

```
新しいセミナーを作りましょう！

まず、あなたの商品やサービスについて教えてください。
お持ちの素材があれば貼り付けてもらうと、一気に情報を読み取れます。

たとえばこんなものです：
  LPのテキスト / メルマガ / セミナー文字起こし / お客様の声
  SNSの投稿 / 商品説明 / お客様とのやりとり

何でもOKです。多いほど精度が上がります。
素材がなければ「ないです」と言ってもらえれば、質問していきますね。
```

3. **素材がある場合:** テキストを受け取り → 情報を読み取り → `save_user_material()` で保存 → 足りない部分だけ追加ヒアリング

4. **素材がない場合:** 対話でヒアリング

5. **ヒアリングで聞くこと:**
   - 何を売ってるか（商品・講座の概要）
   - なぜそれを作ったか（きっかけ・原体験）
   - 大切にしてること
   - お客さんの声（よく言われること）
   - セミナーのゴール（何を売りたいか）
   - セミナータイプ（フロント無料 / バックエンドセールス / 説明会等）
   - 想定時間
   - オファー内容
   - 文体・好みの表現・NG表現

6. **ヒアリングの原則:**
   - 角度を変えて同じテーマを複数回聞く（深い答えを引き出す）
   - 前置きで「なぜ聞いてるか」を伝える
   - 一問一答にしない。対話で自然に深掘り

7. config生成 → ユーザーに確認 → 承認されたら:
   - `create_project(name)` でプロジェクト作成
   - `save_config(project_id, raw_markdown)` で保存

## 禁止事項
- フォーム入力を求めない。全て会話で行う
- ユーザーの承認なしに保存しない
